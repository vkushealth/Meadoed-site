2. Какие данные обрабатываются

Приложение может обрабатывать следующие данные:

  • номер телефона;
  • данные подтверждения входа и технические идентификаторы авторизации;
  • имя профиля, возраст, пол, рост, вес, цель и уровень активности;
  • индивидуальные цели по калориям, белкам, жирам и углеводам;
  • семейные профили;
  • записи дневника питания;
  • продукты, блюда, штрихкоды продуктов и значения КБЖУ;
  • фотографии блюд, если пользователь сам выбирает или делает фото для распознавания;
  • данные дневника питания за день, список продуктов, приемы пищи, количества и значения КБЖУ для AI-аналитики;
  • результаты AI/GPT-распознавания и AI-аналитики, если они сохраняются в приложении или на сервере;
  • технические данные запросов, если они сохраняются сервером: дата и время запроса, IP-адрес, User-Agent, диагностические ошибки;
  • access/refresh tokens, необходимые для авторизации.

3. Зачем обрабатываются данные

Данные используются для:

  • создания и защиты аккаунта пользователя;
  • входа по номеру телефона и одноразовому коду;
  • работы семейных профилей;
  • ведения дневника питания;
  • добавления и распознавания продуктов и блюд;
  • расчета ориентировочных значений КБЖУ;
  • AI/GPT-распознавания блюда по фотографии;
  • составления AI-аналитики питания за день на основе продуктов, блюд и значений КБЖУ пользователя;
  • синхронизации данных между устройством и сервером;
  • поддержки пользователей;
  • защиты от злоупотреблений и технической диагностики.

4. Фотографии блюд

Если пользователь использует распознавание блюда по фотографии, изображение может быть отправлено на сервер приложения и обработано для определения примерного состава блюда и КБЖУ.

Для распознавания фото используется внешний AI/GPT-провайдер, включая OpenAI/GPT, через backend приложения.

5. AI/GPT-обработка

Для функций распознавания блюд и AI-аналитики питания приложение может отправлять данные на сервер приложения, а сервер может передавать часть этих данных внешнему AI/GPT-провайдеру, включая OpenAI.

В AI/GPT-запросы могут входить:

  • выбранные пользователем фотографии блюд;
  • названия продуктов и блюд;
  • количество продуктов, прием пищи, дата и значения КБЖУ;
  • цели питания и профильные параметры, если они нужны для анализа;
  • контекст дневника питания за день для составления AI-аналитики.

AI/GPT-обработка используется только для основных функций приложения: оценки состава блюд, расчета ориентировочных КБЖУ и формирования аналитики питания. Результаты AI могут быть неточными и не являются медицинской рекомендацией.

Обработка данных AI-провайдером регулируется его условиями и политиками. Для OpenAI см. OpenAI Privacy Policy и справку о том, как данные используются для улучшения моделей: openai.com/policies/privacy-policy/, help.openai.com.

Пользователь не должен загружать фотографии, содержащие лишние персональные данные, документы, лица других людей или другую информацию, не нужную для анализа блюда.

6. Передача данных третьим лицам

Данные могут передаваться только в случаях, необходимых для работы приложения, хранения данных, авторизации, распознавания блюд, AI/GPT-аналитики питания, технической поддержки или выполнения требований закона.

К таким третьим лицам могут относиться поставщики backend- инфраструктуры, сервисы отправки одноразовых кодов, сервисы хранения, аналитики и AI/GPT-провайдеры, включая OpenAI.

7. Хранение и защита данных

Данные передаются по защищенному соединению HTTPS. Авторизационные токены хранятся на устройстве пользователя в защищенном хранилище приложения.

Срок хранения данных зависит от срока использования аккаунта. Данные аккаунта и связанные данные удаляются после запроса пользователя, кроме случаев, когда оператор обязан сохранить отдельные сведения по закону, для безопасности или предотвращения злоупотреблений.
